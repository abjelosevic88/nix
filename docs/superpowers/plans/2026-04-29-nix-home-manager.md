# Nix Home-Manager Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a minimal flakes-based home-manager configuration on this Apple Silicon Mac, installed via Determinate Systems' `nix-installer`, tracking nixpkgs `nixos-25.11`.

**Architecture:** Standalone home-manager (no nix-darwin). Single host, `aarch64-darwin`. `~/nix` is a git repo containing `flake.nix`, `home.nix`, and `.gitignore`. Day-to-day edits land in `home.nix`; activation via `home-manager switch --flake ~/nix`.

**Tech Stack:** Nix (upstream, installed via DetSys `nix-installer`), Nix flakes, home-manager `release-25.11`, nixpkgs `nixos-25.11`, zsh.

**Operator note:** Some steps require user interaction (sudo password, installer prompts). Each such step is flagged. There is no automated test framework here — "tests" are concrete shell commands with expected output that verify the system reached the desired state.

**Spec:** `docs/superpowers/specs/2026-04-29-nix-home-manager-design.md`

---

## File Structure

| File | Responsibility |
|---|---|
| `~/nix/flake.nix` | Flake metadata: pin inputs (nixpkgs `nixos-25.11`, home-manager `release-25.11`); expose `homeConfigurations.abjelosevic` output. Rarely edited. |
| `~/nix/home.nix` | The user-facing config: username, home dir, `stateVersion`, `home.packages`, future `programs.*` modules. Edited daily. |
| `~/nix/flake.lock` | Auto-generated by Nix on first build. Pins exact commits of inputs. Committed to git. |
| `~/nix/.gitignore` | Ignores `result`/`result-*` symlinks that some Nix commands leave behind. |

### Task 3: Create `flake.nix`

**Files:**
- Create: `~/nix/flake.nix`

- [ ] **Step 1: Write `flake.nix`**

Create `/Users/abjelosevic/nix/flake.nix` with contents:
```nix
{
  description = "abjelosevic home-manager config";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-25.11";
    home-manager = {
      url = "github:nix-community/home-manager/release-25.11";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { nixpkgs, home-manager, ... }:
    let
      system = "aarch64-darwin";
      pkgs = nixpkgs.legacyPackages.${system};
    in {
      homeConfigurations.abjelosevic = home-manager.lib.homeManagerConfiguration {
        inherit pkgs;
        modules = [ ./home.nix ];
      };
    };
}
```

- [ ] **Step 2: Verify the flake parses**

The flake references `./home.nix` which doesn't exist yet, so a full evaluation will fail. We can still syntax-check by running:
```bash
cd ~/nix
nix flake show 2>&1 | head -5
```
Expected: either succeeds and shows nothing useful yet, OR fails with a message mentioning `home.nix` (file not found / cannot be loaded). Either is fine — we just want to confirm it's not a syntax error in `flake.nix` itself.

If the error is anything else (e.g. "syntax error", "undefined variable") — fix the file before continuing.

- [ ] **Step 3: Stage the file (don't commit yet)**

```bash
cd ~/nix
git add flake.nix
```
We commit after the lock file is generated by the bootstrap step, so they go in together.

### Task 4: Create `home.nix`

**Files:**
- Create: `~/nix/home.nix`

- [ ] **Step 1: Write `home.nix`**

Create `/Users/abjelosevic/nix/home.nix` with contents:
```nix
{ pkgs, ... }:
{
  home.username = "abjelosevic";
  home.homeDirectory = "/Users/abjelosevic";
  home.stateVersion = "25.11";

  programs.home-manager.enable = true;

  home.packages = with pkgs; [

    fd
    bat
    jq
  ];
}
```

- [ ] **Step 2: Verify the flake now evaluates**

Expected: either a clean exit, or warnings only — no errors. (`--no-build` skips actually building anything, just type-checks the config.)

If it fails with a real error (e.g. "attribute missing", "type mismatch") — fix the file before continuing.

- [ ] **Step 3: Stage the file**


### Task 5: Bootstrap home-manager (first activation)

**Operator action required:** This builds packages and may take 30 seconds to a few minutes the first time. May prompt for trust on substituters.

**Files:** none directly modified — generates `~/nix/flake.lock` and creates `~/.config/home-manager/` state.

- [ ] **Step 1: Run the bootstrap command**

```bash
cd ~/nix
nix run github:nix-community/home-manager/release-25.11 -- switch --flake ~/nix#abjelosevic
```

If prompted to trust the substituter `https://cache.nixos.org`, accept (`y`).

Expected: downloads, builds, prints lines like `Activating ...`, ends with something containing `Activation finished successfully` (exact wording may vary, but exit code is 0).

- [ ] **Step 2: Verify generation exists**

```bash
home-manager generations
```
Expected: at least one line listing a generation, e.g.:
```
2026-04-29 12:34 : id 1 -> /nix/store/...-home-manager-generation
```
If the `home-manager` command isn't on PATH yet, open a new terminal and try again — activation adds it via `programs.home-manager.enable = true`.

- [ ] **Step 3: Verify a managed package is on PATH**

```bash
which ripgrep
```
Expected: a path under `/nix/store/.../bin/ripgrep` (NOT `/usr/bin/...` or `/opt/homebrew/...`).

```bash
ripgrep --version
```
Expected: prints a version string like `ripgrep 14.x.x`.

- [ ] **Step 4: Verify `flake.lock` was generated**

```bash

```
Expected: file exists, non-empty.

```bash

```
Expected: `flake.lock` shows as untracked.
